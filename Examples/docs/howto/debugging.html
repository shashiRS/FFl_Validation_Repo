Debugging#

Debug a test case implementation#

This demos demonstrate how to call your test case implementation and process it with a couple of bsigs and produce a report on the processed results.

This is very useful during development since it won’t require access to the oracle database or HPC.

All you need is a couple of processing outputs (e.g. BSIGs) with function output of the component you want to test.

 1def main(data_folder: Path, temp_dir: Path = None, open_explorer=True):
 2    """Optional, call to debug to set up debugging in the simplest possible way.
 3
 4    When calling the test case you need to provide a valid input to
 5    execute the test (e.g. a BSIG file) and report the result.
 6
 7    This is only meant to jump start testcase debugging.
 8    """
 9    test_bsigs = [data_folder / f"test_input_{k}.bsig" for k in range(3)]
10    os.makedirs(data_folder, exist_ok=True)
11    for b in test_bsigs:
12        generate_bsig(b)
13
14    debug(
15        [
16            ExampleMinimalTestCase,
17        ],
18        *test_bsigs,
19        temp_dir=temp_dir,
20        open_explorer=open_explorer,
21        kpi_report=False,
22        dev_report=True,
23    )
24    _log.debug("All done.")

The full example can be found in examples/debug/single_bsig.py.
