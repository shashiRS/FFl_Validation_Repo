Quickstart¶

Prerequisites¶

You’ll need to know a bit of Python. To work the examples you need install a venv with adas-tsf and its dependencies installed.

Learner profile¶

This is a quick overview of test implementation and processing with the TSF library. You should be familiar with pandas Dataframes and software testing.

Learning Objectives¶

After reading, you should be able to:

  • What a test case and test step is.

  • Know how to map signals

  • Know how to setup the runner

Testscripting with TSF¶

TSF provides a framework to support test implementation and processing for KPIs. The main purpose is the processing of Simulation outputs (e.g. HIL, SIL, VSP) and evaluating the output signals on certain behaviors. The number of processed outputs can vary from one up to many thousand.

The outputs are processed in batch without depending on each other in either input order or implementation order. With that it is possible to develop and test the test-script implementation on a single input, as shown in this quickstart.

Let’s assume we have some SIL Outputs of an EBA CCRs use-case.

Use-case parameters:

  • Approach ego velocity: 50 km/h

  • Target Overlap: 100%

  • Target rotation: 0°

Evaluation Scope:

  • Assert that the EBA function outputs a prebrake deceleration signal.

  • Assert the reaction is early than 1.1 s TTC

Have a look at parts which interact during the evaluation. First the TSF will open the relevant inputs, the signals which need to be read are defined in the signal definition. After all signals are read, the test step are processed. For each input of each test-step a result has to be computed. Additionally, you may want to add more informal data and statistics which should be reported later on.

Setting up Testcases and Teststeps¶

Lets start with implementing a testcase. Test-cases define the inputs which are valid for the given test-case and its test-steps. The test case verdict is computed from the verdicts of its test steps. If one or more test-steps are failed the entire testcase will be marked as failed.

 1@testcase_definition(
 2    "EX_EBA_000_001",
 3    "Example Usecase Test implementation",
 4    "This example demonstrates how to process carmaker erg files within the TSF test scripting framework.",
 5)
 6@register_generic_collection("Example ERG files", discipline="VSP")
 7class ExampleUsecaseTestCase(TestCase):
 8    """Example test case."""
 9
10    @property
11    def test_steps(self):
12        """Define the test steps."""
13        return [ExampleAEBTestStep]

Test-step will be used to compute result for the read data. You need to define the needed signals by providing a signal definition.

 1@teststep_definition(
 2    2,
 3    "AEB activation check",
 4    "This example test step demonstrates how to check for a signal activation which is read from an erg file.",
 5    ExpectedResult(0.3, unit="s", operator=RelationOperator.GREATER_OR_EQUAL, aggregate_function=AggregateFunction.ALL),
 6)
 7@register_signals(EBA, EbaSignals)
 8class ExampleAEBTestStep(TestStep):
 9    """Example test step."""
10
11    def process(self, **kwargs):
12        """Check AEB for a falling flank and validation TTC at that point in time."""
13        # Get the read data frame
14        eba = self.readers[EBA]
15
16        # compute the TTC:
17        # CCRs: relative distance / ego velocity
18        ttc = (eba[EbaSignals.Columns.TGT_X] - eba[EbaSignals.Columns.EGO_X]) / eba[EbaSignals.Columns.EGO_VELO]
19
20        # Filter unreasonable datas and replace with 40:
21        #   * TTC > 40
22        #   * TTC < 0
23        #   * Ego is hardly moving
24        lim = (ttc > 40) | (ttc < 0) | (eba[EbaSignals.Columns.EGO_VELO] < 0.1)
25        ttc.loc[lim] = 40
26
27        eba["TTC"] = ttc
28
29        # check for falling flanks
30        flanks = eba[EbaSignals.Columns.PRE_BRAKE_DECELL].diff() > 0
31
32        # slice dataframe to the falling flank(s)
33        activation = eba.loc[flanks]
34
35        if not activation.empty:
36            # Evaluate TTC at the timestamp of the falling flank.
37            ttc = activation["TTC"].values[0]
38
39            # Store the result
40            self.result.measured_result = Result(ttc, unit="s")
41        else:
42            # Store NaN when no activation could be found.
43            self.result.measured_result = NAN

A signal definition defines the mapping between the signal name from the input and the column in the read dataframe. The read signals are provided to the test steps as a dataframe by it alias.

 1class EbaSignals(SignalDefinition):
 2    """Signal mapping for EBA."""
 3
 4    class Columns(SignalDefinition.Columns):
 5        """Definition of the dataframe columns."""
 6
 7        PRE_BRAKE_DECELL = "pre_brake_decell"
 8        EGO_X = "ego_x"
 9        TGT_X = "tgt_x"
10        EGO_VELO = "ego_velo"
11
12    def __init__(self):
13        """Initialize the signal definition."""
14        super().__init__()
15
16        self._properties = {
17            self.Columns.PRE_BRAKE_DECELL: "ARS540 Device.AlgoSenCycle.MEDIC_PreBrakeMeas.fPreBrakeDecel",
18            self.Columns.EGO_X: (
19                "OSI Data Processor.OSI_DATA.sensor_view.global_ground_truth.moving_object[0].base.position.x"
20            ),
21            self.Columns.TGT_X: (
22                "OSI Data Processor.OSI_DATA.sensor_view.global_ground_truth.moving_object[1].base.position.x"
23            ),
24            self.Columns.EGO_VELO: (
25                "OSI Data Processor.OSI_DATA.sensor_view.global_ground_truth.moving_object[0].base.velocity.x"
26            ),
27        }

Debugging the test implementation¶

During the development phase it is useful to run a single testcase with a single input without any complex setup. This can be accomplished by calling the test case with debug(..).

Debugging for the implementation above would look like:

 1def main(temp_dir: Path = None, open_explorer=True):
 2    # Entry point for debugging during development.
 3    sample_input = (
 4        Path(__file__).parent.parent / "data" / "tutorial" / "CCRs~V_VUT=50.0~OVERLAP=100.0~Z_ROTATION=0.0.bsig"
 5    )
 6
 7    debug(
 8        ExampleUsecaseTestCase,
 9        sample_input,
10        temp_dir=temp_dir,
11        open_explorer=open_explorer,
12    )
